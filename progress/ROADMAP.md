# karpathy-kwiki 路线图

**最后更新**: 2026-04-16

## 项目愿景

打造零门槛的个人知识库自动入库工具：输入链接/文件，AI 自动提炼、整理、入库到 Obsidian，并可持续查询和质检。

## 各阶段目标

### v0.1.0 ✅ 基础架构（已完成）
- Electron + React + Vite 项目初始化
- 主进程 IPC 文件系统操作
- 6 个功能页面
- Tkinter GUI 主入口（`kwiki.exe`）

### v0.2.0 ✅ 核心功能（已完成）
- **Ingest**：网页/视频/PDF/Word → AI 提炼 → 存入 wiki/
- **Query**：向知识库提问，AI 综合回答，评估缺口
- **Lint**：扫描 wiki/，发现矛盾/过时/缺失链接，质量评分

### v0.3.0 🔄 Query 增强（进行中）
关联 epic-003

- **功能 1：Query 存入 Wiki**
  - LLM 质量评估（分数 + 建议）
  - 前端"保存到 Wiki"按钮
  - 存入时带 source: query-generated + derived_from

- **功能 2：话题推荐**
  - A. 知识缺口推荐（缺失链接/orphan/过时）
  - B. 相关问题推荐（基于查询/浏览）
  - C. 知识探索（随机推荐未深入主题）

### v1.0.0 📋 正式发布（计划中）
- 文档完善
- 安装包优化
- 自动更新机制
- **技术栈整合**：
  - 将 Electron 版本的现代化界面与 Python 版本的稳定功能整合
  - 统一技术栈，提高代码可维护性
  - 优化性能和用户体验
