## 背景

KWiki 需要一个进度管理系统来跟踪任务、记录决策和管理开发流程。

## 决策

**我们决定**：采用分层 Markdown 文件 + Obsidian 管理项目进度。

## 方案选项

### 选项 A：单一 Markdown 文件 ❌
**描述**：所有进度放在一个 `修改.md` 文件中  
**缺点**：文件越来越大，难以定位，历史不清晰

### 选项 B：GitHub Issues / Projects ❌
**描述**：使用 GitHub 项目管理  
**缺点**：需要网络，依赖 GitHub 平台

### 选项 C：分层 Markdown + Obsidian ✓（当前选择）
**描述**：多个 Markdown 文件配合 Obsidian 可视化  
**优点**：
- 文件结构清晰
- 支持双向链接和图谱视图
- 本地优先，不依赖网络
- 与项目理念一致（KWiki 就是 Obsidian 工具）

## 后果

### 正面
- 进度管理规范化和可视化
- 历史记录和决策追踪清晰
- 可纳入 Git 版本控制

### 中立
- 需要使用 Obsidian 查看图谱关系
