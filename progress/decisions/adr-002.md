## 背景

KWiki 需要一个跨平台桌面 GUI，Tkinter 适合 Python 层，Electron + React + Vite 提供更好的前端体验。

## 决策

**我们决定**：Electron + React + Vite 作为前端技术栈，Tkinter 作为备用/轻量方案。

## 方案选项

### 选项 A：纯 Tkinter ❌
**描述**：纯 Python GUI  
**缺点**：前端体验差，不适合复杂交互

### 选项 B：Electron + React + Vite ✓（当前选择）
**描述**：成熟桌面框架 + 现代前端  
**优点**：
- React 组件化开发效率高
- Vite 快速热更新
- Electron 成熟稳定（VS Code、Slack 等都在用）
- 支持 Windows/macOS 多平台

### 选项 C：Tauri ❌
**描述**：Rust 后端 + Web 前端  
**缺点**：生态不如 Electron 成熟，配置成本高

## 后果

### 正面
- 前端开发体验好，HMR 热更新
- React 生态丰富（React Router、状态管理等）

### 负面
- 包体积较大（ Electron ~150MB）
- 需要 Node.js 环境

### 中立
- Tkinter GUI（`kwiki.py`）作为轻量备选入口保留
