# Karpathy LLM Wiki 项目计划与行为规范

## 一、项目概述

### 1.1 项目背景
基于 Andrej Karpathy 提出的 LLM Wiki 概念，构建一个本地知识库管理系统，结合 Obsidian 进行知识可视化，并支持本地 AI 后端（Ollama 和 LM Studio）进行知识处理和维护。

### 1.2 项目目标
- 实现 Karpathy LLM Wiki 的三层架构
- 支持三个核心操作：Ingest（摄入）、Query（查询）、Lint（检查）
- 与 Obsidian 无缝集成，提供知识图谱可视化
- 支持本地 AI 后端配置，确保数据隐私和离线使用
- 提供直观的用户界面，简化知识管理流程

## 五、架构说明

### 5.1 三层架构
```
raw/          → 原始资料（只读）
wiki/         → 结构化知识页面
outputs/      → 可视化输出
```

### 5.2 三个核心操作
```
Ingest        → 知识摄入和提炼
Query         → 知识查询和推理
Lint          → 质量检查和维护
```
